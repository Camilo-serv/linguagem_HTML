Originalmente fortemente identificada pela sua característica hipermidiática, a linguagem HTML tem se tornado cada vez mais robusta. Em sua atual versão, HTML 5, já é possível construir aplicações WEB com armazenamento local, desenho 2D, uso de sockets e threads etc.

Utilizando a organização estabelecida pelo site Mozilla Web Docs (https://developer.mozilla.org/pt-BR/docs/Web/HTML/HTML5) para a apresentação dos recursos oferecidos HTML 5, podemos apresentá-los da seguinte maneira:

Semântica: permite você descrever mais precisamente o seu conteúdo.

Conectividade: permite uma comunicação com o servidor de formas modernas e inovadoras.

Offline e armazenamento: Permite que páginas web armazenem dados localmente do lado do cliente e opere de forma offline mais eficientemente.

Multimídia: Viabiliza a utilização de áudio e vídeo de forma primorosa na Web Aberta.

Gráficos e efeitos 2D/3D: viabiliza um leque diversificado de opções de representação gráfica.

Performance e integração: fornece grande otimização de velocidade e melhor utilização do hardware do computador.

Acesso ao dispositivo: viabiliza a utilização de diversos métodos e dispositivos de entrada e saída.

Estilização: permite aos autores a escrita de temas mais sofisticados.

Sendo a versão atual da linguagem HTML, o HTML 5 vem facilitar a manipulação de elementos, dotando os desenvolvedores de maior facilidade para modificar as características de objetos. Desta forma, ela fornece ferramentas para que tanto CSS e JavaScript possam fazer seu trabalho de forma mais eficiente, através de API's disponibilizadas.

A nova versão também traz novas tags frente a versões anteriores e modifica a função de algumas outras. Por exemplo, as versões antigas não continham um padrão para a criação de seções específicas como rodapé, cabeçalho, menus etc. Não eram identificados através de id's, classes ou mesmo tags específicas, o que ocorre no HTML 5.

Atualmente, os principais navegadores utilizados na internet já oferecem suporte ao HTML5, o que torna mais fácil a vida dos desenvolvedores desta tecnologia ao não se preocuparem com questões de compatibilidade dos recursos com os browsers.
